Files Created

1. utils/gmail_auth.py — Gmail OAuth Module

  • gmail_auth_url() — Builds Google OAuth consent URL (direct URL, no PKCE)
  • gmail_exchange_code(code, state) — Exchanges auth code for tokens via requests.post
  • get_gmail_service() — Returns authorized Gmail API v1 service (auto-refreshes tokens)
  • is_gmail_connected() — Checks if valid gmail_token.json exists
  • get_gmail_account_info() — Returns connected email from Gmail profile

2. utils/ash_scanner.py — Email Scanner

  • scan_inbox(max_results, days_back) — Searches Gmail for supplier keywords
  • classify_email(subject, snippet) — Regex classification: invoice/shipment/delay/tracking/etc
  • match_supplier(sender, sender_email, suppliers) — Fuzzy match via SequenceMatcher (60%)
  • match_order(subject, snippet, orders) — Matches order IDs/PO numbers in text

3. routes/ash.py — Flask Blueprint

GET /api/ash/scan — Scan inbox for supplier emails (params: max_results, days_back)
GET /api/ash/status — Gmail connection status
GET /oauth/gmail/start — Redirects to Google OAuth consent
GET /oauth/gmail/callback — Handles OAuth callback, saves token

4. gmail_credentials.json — OAuth Web Client

Client "Ash Gmail Scanner" (Web type), redirect URI: http://localhost:5000/oauth/gmail/callback

5. gmail_token.json — Auto-generated after OAuth

Contains access_token, refresh_token, auto-refreshes when expired

Key Fix: PKCE Error

Original implementation used google_auth_oauthlib.flow.Flow which adds PKCE (code_verifier) by default, causing "Missing code verifier" errors. Rewrote gmail_auth.py to use direct requests-based OAuth — manually constructing the auth URL and exchanging the code via HTTP POST. No PKCE, no code verifier, clean flow.

Latest Change: days_back Default → 30

Changed routes/ash.py line 38: default days_back from 7 to 30 so the scan covers a full month by default. With 7 days the inbox returned 0 results; with 30 days it found 34 emails (invoices, tracking, delays).
